**Reference:** Department of Home Affairs (Cth), *Code of Practice: Securing the Internet of Things for Consumers* (2020).

### Overview
The *Code of Practice: Securing the Internet of Things for Consumers* is a voluntary set of measures recommended by the Australian Government as a minimum security standard for consumer IoT devices. Developed by the Department of Home Affairs in partnership with the Australian Signals Directorate (ASD), it aims to improve the "security by design" of smart devices (e.g., smart TVs, home assistants, wearables) to protect against malicious cyber activity.

### Key Principles
The Code comprises 13 principles, with the first three identified as the highest priority for immediate security benefits:

1.  **No duplicated default or weak passwords:** Passwords must be unique, complex, and not resettable to a common factory default.
2.  **Implement a vulnerability disclosure policy:** Manufacturers and service providers should provide a public point of contact for reporting security issues.
3.  **Keep software securely updated:** Software and firmware should be securely updateable, with clear end-of-life policies regarding support duration.
4.  **Securely store credentials:** Avoid hard-coded credentials in software or hardware.
5.  **Ensure personal data is protected:** Compliance with the *Privacy Act 1988* (Cth) and Australian Privacy Principles (APPs); encryption of data at rest and in transit.
6.  **Minimise exposed attack surfaces:** Operate on the "principle of least privilege" and disable unused functionality.
7.  **Ensure communication security:** Use encryption for data requiring confidentiality or integrity protection.
8.  **Ensure software integrity:** Use secure boot mechanisms to verify firmware.
9.  **Make systems resilient to outages:** Ensure local functionality persists during network or power loss where possible.
10. **Monitor system telemetry data:** Monitor usage/measurement data for security anomalies.
11. **Make it easy for consumers to delete personal data:** Provide clear instructions for factory resets and data deletion upon transfer of ownership.
12. **Make installation and maintenance easy:** Follow best practices for security and usability during setup.
13. **Validate input data:** Ensure data from APIs and user interfaces conforms to expectations.

### Significance
*   **Regulatory Approach:** Represents Australia's "soft law" approach to IoT security, opting for voluntary industry standards rather than immediate mandatory regulation, though it signals government expectations.
*   **International Alignment:** Aligns with international standards (notably the UK's Code of Practice) to ensure consistency for global manufacturers operating in the Australian market.
*   **Consumer Protection:** Addresses the systemic issue of security being an "afterthought" in IoT development, aiming to build consumer confidence and mitigate risks to national security and the economy.
