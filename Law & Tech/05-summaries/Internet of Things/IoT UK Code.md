# Code of Practice for Consumer IoT Security (2018)

**Reference:** Department for Digital, Culture, Media and Sport (DCMS) & National Cyber Security Centre (NCSC), *Code of Practice for Consumer IoT Security*, October 2018.

## Overview / Key Points

The *Code of Practice for Consumer IoT Security* is a set of thirteen outcome-focused guidelines designed to improve the cyber security of consumer Internet of Things (IoT) products and associated services. It aims to ensure products are "secure by design" and to protect consumer privacy and safety.

### The 13 Guidelines
The Code prioritises the first three guidelines as they offer the most immediate security benefits:

1.  **No default passwords**: All IoT device passwords must be unique and not resettable to a universal factory default.
2.  **Implement a vulnerability disclosure policy**: Companies must provide a public point of contact for security researchers to report issues, which must be acted upon in a timely manner.
3.  **Keep software updated**: Software must be securely updateable, and companies must publish an end-of-life policy stating the minimum support period for updates.

The remaining guidelines include:
4.  **Securely store credentials**: No hard-coded credentials; use secure storage mechanisms.
5.  **Communicate securely**: Use encryption for security-sensitive data in transit.
6.  **Minimise exposed attack surfaces**: Follow the "principle of least privilege" (close unused ports, etc.).
7.  **Ensure software integrity**: Use secure boot mechanisms and alert users to unauthorised changes.
8.  **Protect personal data**: Adhere to data protection laws (e.g., GDPR) and provide transparent information on data usage.
9.  **Make systems resilient to outages**: Ensure local functionality during network or power loss.
10. **Monitor system telemetry**: Monitor for security anomalies while minimising personal data collection.
11. **Easy deletion of personal data**: Allow users to wipe data for transfer of ownership or disposal.
12. **Easy installation and maintenance**: Reduce consumer confusion through good UI/UX and clear guidance.
13. **Validate input data**: Prevent subversion via incorrectly formatted data or code.

## Significance

This Code of Practice is significant as a foundational regulatory instrument in the "Law & Tech" space for several reasons:

*   **Shift to "Secure by Design"**: It moves the burden of security from the consumer to the manufacturer and service provider, advocating for security to be integrated throughout the product lifecycle.
*   **Harmonisation with Data Protection**: It explicitly links IoT security to compliance with the GDPR and Data Protection Act 2018, noting that secure processing is a legal requirement.
*   **Global Influence**: Developed by the UK government (DCMS and NCSC), it has served as a template for international standards (such as ETSI EN 303 645) and future mandatory regulations.
*   **Addressing Systemic Risk**: By mitigating poorly secured devices, it reduces the threat of Distributed Denial of Service (DDoS) attacks that can impact the wider internet infrastructure.
